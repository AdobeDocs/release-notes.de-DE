### Fehlerbehebungen in Adobe Analytics und Customer Journey Analytics {#aa-fixes}

* Es wurde ein Analysis Workspace-Problem behoben, bei dem die Zielgruppen-ID in Dimensionselementen fehlte. (AN-262038; AN-279315)
* Fehlerkorrektur - Benutzer können jetzt gespeicherte [!DNL Target] Projekt in Workspace. (AN-277461) AN-275825; AN-266397)
* Es wurde ein Problem behoben, bei dem nicht aktivierte Funktionen in der Benutzeroberfläche sichtbar sind. (AN-262006)
* Es wurde ein Problem behoben, das beim Ändern des Datums mithilfe des Datumsfelds in Workspace auftrat. Dies führte zu der [!UICONTROL Endzeit] Wechsel von 11:59 Uhr zu 12:00 Uhr. (AN-277269; AN-277481)
* Es wurde ein Fehler behoben, der dazu führte, dass die Segment-Benutzeroberfläche beim Hinzufügen neuer Segmente zu einem bereits geladenen Segment fehlschlug. (AN-260827)
* Es wurde ein Problem behoben, durch das Benutzer nicht auf freigegebene Workspace-Projekte zugreifen können. (AN-267529)
* Es wurde eine Fehlermeldung hinzugefügt, die anzeigt, wann ein rollierender Datumsbereich ein Startdatum nach dem Enddatum hat. (AN-270488)
* Es wurden verschiedene Probleme mit Daten-Feeds behoben. (AN-275876) AN-270512; AN-277284; AN-277290 AN-274893; AN-274606; AN-269651)
* Es wurde ein Problem behoben, durch das Datumsbereiche in Diagrammen Datumsfilter in Tabellen ignorierten. (AN-263999)
* Es wurde ein Problem behoben, bei dem terminierte Berichte aufgrund der Sommerzeit frühzeitig gesendet wurden. (AN-276410; AN-276305)
* Es wurde ein Problem mit dem Projekt-Download behoben, bei dem `.csv` -Datei in Workspace fehlschlägt. (AN-275834)

   * **Experience Manager Assets as a Cloud Service**

      _Neue Funktion_

      * Dynamic Media [!UICONTROL Smartes Zuschneiden von Bildern] und [!UICONTROL Muster] basiert nun auf den neuesten Sensei-Diensten, die verbesserte Ernten- und Farbmuster erzeugen. Außerdem wurde eine Erweiterung gestartet, um unterschiedliche Zuschnittinhalte für dasselbe Seitenverhältnis, aber über verschiedene Auflösungen hinweg zu generieren.       _Neue Funktionen im Experience Manager Assets-Vorabversionskanal_

      * Dynamic Media - Sie können jetzt die Experience Manager Dynamic Media-Benutzeroberfläche zum Konfigurieren von [!UICONTROL Allgemeine Einstellungen] und [!UICONTROL Veröffentlichungseinstellungen] anstatt das Dynamic Media Classic-Desktop-Programm durchlaufen zu müssen.

Sie können jetzt die [Vorgänge aktualisieren, löschen, umbenennen und verschieben](https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/assets/admin/use-assets-across-connected-assets-instances.html?lang=en) auf Remote-DAM-Assets oder -Ordnern. Die Aktualisierungen sind mit einiger Verzögerung automatisch in der Sites-Bereitstellung verfügbar.
   * **Experience Manager Forms as a Cloud Service**

      _Neue Funktion_

      * **Experience Manager-Workflow-Daten für sichere Verarbeitung externalisieren**: Sie können in verarbeiteten Experience Manager-Workflows Daten speichern (Experience Manager [!UICONTROL Workflow-Variablen] -Daten), die SPD -Elemente (Sensible Personal Data) in einem kundenverwalteten Repository zur sicheren Verarbeitung enthält. Die Datenelemente und Workflow-Variablen werden nicht im Experience Manager-Repository gespeichert und bei der Verarbeitung des Workflows bei Bedarf aus einem kundenverwalteten Repository abgerufen.

      _Neue Funktionen im Experience Manager Forms-Vorabversionskanal_

      * **AEM Forms as a Cloud Service - Kommunikation**: [Kommunikations-APIs](https://experienceleague.adobe.com/docs/experience-manager-cloud-service/content/forms/using-communications/aem-forms-cloud-service-communications.html?lang=en) hilft Ihnen, eine Vorlage und XML-Daten zu kombinieren, um Druckdokumente in verschiedenen Formaten zu generieren. Mit dem Dienst können Sie Dokumente im synchronen Modus und im Batch-Modus generieren. Mit den APIs können Sie Anwendungen für folgende Aufgaben erstellen:
         * Erzeugen von Dokumenten durch Ausfüllen von Vorlagendateien (PDF und XDP) mit XML-Daten.
         * Erstellen von Ausgabeformularen in verschiedenen Formaten, einschließlich nicht interaktiver PDF-Printstreams.

Dies hilft einem Site-Besucher, alle verfügbaren Formulare zu finden. Darüber hinaus kann der Besucher [!UICONTROL Forms Portal] , um den Entwurf eines adaptiven Formulars zu speichern und darauf zuzugreifen und sich die PDF-Version eines gesendeten adaptiven Formulars anzusehen.
