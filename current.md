### Fehlerbehebungen in Adobe Analytics {#aa-fixes}

* Es wurde ein Problem behoben, bei dem die Berichte-API keine aktuellen Metrikwerte zurückgab. (AN-225617)
* Es wurde ein Fehler behoben, der verhinderte, dass [!UICONTROL Classification-Regeln] Daten für [!UICONTROL Marketing-Kanal-Details]klassifizieren konnten. (AN-224832)
* Es wurde ein Fehler behoben, der beim Erstellen neuer Projekte in einer _Virtual Report Suite_ den Fehler &quot; [!UICONTROL Fehlende Komponenten]&quot;verursachte.(AN-226808)
* Es wurde ein Fehler behoben, der beim Kuratieren einer Virtual Report Suite den Fehler &quot; _Fehlende Komponenten_ &quot;verursachte. (AN-228257)
* Es wurde ein Fehler behoben, der die Erstellung neuer [!UICONTROL Reports &amp; Analysen] -Zielgruppen und Kalenderdaten verhinderte. (AN-224872, AN-224890, AN-224914, AN-226661)
* Es wurde ein Problem behoben, das zu fehlenden Aktivitäten im A4T-Bedienfeld in [!UICONTROL Workspace]führte. (AN-224606)
* Es wurde ein Problem mit Duplikat-Treffern in [!UICONTROL Data Feeds]behoben. (AN-226308)
* Fixed an issue where calculated metrics with participation attribution did not return proper values. (AN-224642, AN-225190)
* Es wurde ein Problem behoben, bei dem Segmentdaten, die freigegeben wurden, von mehr als drei Tagen in [!DNL Analytics] Anspruch genommen wurden, um in angezeigt zu werden [!DNL Audience Manager] [!DNL Audience Manager].(AN-226649)
* Fixed an issue with not being able to use the [!UICONTROL Analyze Further] link in [!UICONTROL Intelligent Alerts] emails. (AN-226823)
* Es wurde ein Fehler behoben, der dazu führte, dass keine Segmente in einer [!UICONTROL Virtual Report Suite]erstellt werden konnten. (AN-227039)
* Es wurde ein Problem behoben, bei dem intelligente Warnhinweise nicht bearbeitet werden konnten. (AN-227162)

### **Produktversionen**

* **AEM as a Cloud Service**

   What is new on AEM as a Cloud Service? Zu den hervorgehobenen Highlights zählen:

   * AEM Commerce ist jetzt auf dem Cloud Service verfügbar. See [Getting started with AEM Commerce as a Cloud Service.](https://docs.adobe.com/content/help/en/experience-manager-cloud-service/commerce/getting-started.html)
   * Connectors for Adobe Target and Adobe Analytics enhancements includes user interface improvements, Classic UI replacement, and Adobe Launch integration. See [Integrating Adobe Analytics](https://docs.adobe.com/content/help/en/experience-manager-cloud-service/sites/integrations/integrating-adobe-analytics.html) and [Integrating Adobe Target.](https://docs.adobe.com/content/help/en/experience-manager-cloud-service/sites/integrations/integrating-adobe-target.html)
   * Asset Compute Service is a scalable and extensible service to process assets. Administratoren können Experience Manager so konfigurieren, dass mit dem Asset Compute-Dienst erstellter benutzerdefinierter Arbeiter aufgerufen wird. Developers can use the service to create specialized custom workers that cater to complex use cases. This web service can generate thumbnails for different file types, high-quality image renderings from Adobe file formats, encode videos (future), extract metadata, extract full text as precursor for indexing, and run an asset through all available Sensei services. Siehe [Verwenden von Asset-Mikrodiensten und Profilen zur Verarbeitung.](https://docs.adobe.com/content/help/de-DE/experience-manager-cloud-service/assets/manage/asset-microservices-configure-and-use.html)
   * [Asset Compute Service](https://docs.adobe.com/content/help/en/asset-compute/using/home.html) is made available to be extended to create custom application. Es handelt sich um einen skalierbaren und erweiterbaren Dienst [!DNL Adobe Experience Cloud] zur Verarbeitung digitaler Assets. It can transform image, video, document, and other file formats into different renditions including thumbnails, extracted text and metadata, archives and more. Developers can create custom applications (also called custom workers) to address custom use cases. Es wurde mit [Project Firefly](https://www.adobe.io/apis/experienceplatform/project-firefly/docs.html) gebaut und arbeitet in serverless [Adobe I/O Runtime](https://www.adobe.io/apis/experienceplatform/runtime.html).
