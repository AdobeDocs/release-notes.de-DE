### [!UICONTROL Media Analytics SDK] für [!DNL iOS] und [!DNL Android] (ehemals VHL SDK){#media-analytics-sdk}

| Funktion | Beschreibung |
| -----------| ---------- |  
| [!UICONTROL Adobe Analytics für Audio] | Verbindet die Audio-Interaktion eines Zuhörers bei Streams mit dem gesamten digitalen Verhalten. Somit können Sie verstehen, wer wo, wann und wie häufig zuhört, und das Benutzerverhalten auf dem üblicherweise fragmentierten Audio-Marktplatz auswerten. Diese Funktion ermöglicht es Unternehmen, die Reichweite ihrer Zuhörer sowie die Beliebtheit ihrer Inhalte zu messen, und bietet Einblicke in die Möglichkeiten, die weitere Interaktion der Anwender basierend auf ihren Verhaltensweisen und vorgeschlagenen ähnlichen Themen zu gewährleisten. Messen Sie 10-Sekunden-Audioinhalte und 1-Sekunden-Streaming-Anzeigen mit unseren erstklassigen SDKs, um sowohl die Qualität des Erlebnisses (Anhalten, Fehler, Zeit bis Beginn) als auch den Inhalt und KPIs zu bewerten. |
| Werbepausen verwalten | Verbesserte Anzeigenverfolgung in Szenarien, in denen die SDK-Logik standardmäßig wieder zur Inhaltswiedergabe zurückkehrt, wenn ein Player einen Werbeunterbrechung eingegeben hat und für einen bestimmten Zeitraum keine Aufrufe gesendet werden, selbst wenn sich der Player noch in einem Werbeunterbrechungszustand befindet. Während einer Pre-Roll-Anzeige wird möglicherweise ein Content-Beginn ausgelöst, bevor die Werbeunterbrechung abgeschlossen ist. In Media 2.2 SDK werden Pufferaufrufe innerhalb einer Werbeunterbrechung zwischen Anzeigen gesendet, wenn es eine lange Verzögerung zwischen Anzeigen gibt. |
| SDK-Namensänderung | Ab Version 2.2 werden die Video Heartbeat Library (VHL)-SDKs in Media SDK umbenannt. Das Media 2.2 SDK ist vollständig rückwärtskompatibel mit der VHL 2.X SDK-Serie. Die Namensänderung stellt keine funktionale Umgehung dar, sondern lediglich eine Änderung der Namenskonvention. |

**Fehlerbehebungen, Verbesserungen und veraltete Funktionen**

Der Name des Berichts über den Verlauf des Ausgangs wurde auf den Bericht über den Verlauf der ausgehenden Datei aktualisiert. Der vorherige Name veranlasste einige Kunden zu der Annahme, dass der Bericht ausgehende Daten für HTTP-Ziele anzeigen würde. Tatsächlich umfasst der Bericht Dateien, die an S3- oder FTP-Standorte gesendet werden.

**Bekannte Probleme**

* Die aktuelle Version von Safari bietet Funktionen für ITP 2.0 (Intelligent Tracking Prevention). Diese betreffen die Kennzahlen zur ansprechbaren Zielgruppe für Ihre Safari-Anwender und die Datenerfassung über das h_referer-Signal. Erfahren Sie mehr über Safari-Traffic als [Grund für geringe Trefferquoten bei ansprechbaren Zielgruppen](https://marketing.adobe.com/resources/help/en_US/aam/addressable-audiences-match-rates.html) und die [Datenerfassung mit dem h_-Präfix](https://marketing.adobe.com/resources/help/en_US/aam/r_tb_variable_prefixes.html).
* Durch die Einführung von [Eigenschaftenausschlüssen bei der Algorithmus-Modellierung](https://marketing.adobe.com/resources/help/en_US/aam/trait-exclusion-algo-models.html) kam es bei Kunden, die [rollenbasierte Zugangssteuerung](https://marketing.adobe.com/resources/help/en_US/aam/c_administration.html) (Role-Based Access Controls, RBAC) verwenden, zu Problemen. Wenn Sie beim Erstellen eines neuen Modells nur Datenquellen auswählen, auf die Sie Zugriff haben, können Sie die entsprechenden Eigenschaften im Fenster **Ausnahmen** anzeigen. Wenn Sie jedoch neben den Datenquellen, auf die Sie Zugriff haben, auch andere Datenquellen auswählen, wird eine leere Liste angezeigt. Heben Sie die Auswahl der Datenquellen auf, auf die Sie keinen Zugriff haben, um die Eigenschaften anzuzeigen. (AAM-42380)
